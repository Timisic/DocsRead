# 核心原则：查阅 Docs

这是使用 OpenClaw 最重要的原则，必须牢记。

---

## 为什么必须查阅 Docs？

OpenClaw 是一个非常灵活的系统，你可以修改它的提示词、配置、Skills 等几乎所有东西。这种高自由度是一把双刃剑。

### 好处

- 可以完全定制你需要的功能
- 能适应各种特殊场景

### 风险

- 不当的配置可能导致系统崩溃
- 配置越改越复杂，最后难以维护
- AI 可能产生幻觉，生成不存在的配置项

---

## 如何正确配置

### 错误示范

```
"帮我配置一个流式输出功能"
"帮我优化一下配置"
"帮我添加一个新功能"
```

### 正确示范

```
"请查阅 /docs/ 下关于流式输出的文档，然后按照文档说明帮我配置流式输出功能"
"请查阅 /docs/config/ 下关于性能优化的文档，告诉我有哪些可以优化的配置项"
"请查阅 /docs/features/ 下关于 XXX 的文档，帮我实现这个功能"
```

---

## 配置前的检查清单

在配置任何功能之前，请确保：

- [ ] 告诉 OpenClaw 查阅相关 Docs
- [ ] 确认 AI 理解了文档内容
- [ ] 询问配置的具体步骤
- [ ] 逐步执行，每一步都确认成功
- [ ] 测试配置是否正常工作

---

## OpenClaw 文档结构

OpenClaw 的官方文档位于 `/docs/` 目录下，主要包含：

| 目录 | 内容 |
|-----|------|
| `/docs/channels/` | 各聊天平台的配置文档 |
| `/docs/skills/` | Skills 的使用和配置文档 |
| `/docs/config/` | 配置项的详细说明 |
| `/docs/features/` | 各功能的实现方法 |
| `/docs/api/` | API 接口文档 |

---

## 常见错误及避免方法

### 错误 1：直接让 AI 配置

```
"帮我配置一个 Cron 任务"
```

**正确做法**：
```
"请查阅 /docs/ 下关于 Cron 任务的文档，然后帮我配置一个每天早上 8 点执行的任务"
```

### 错误 2：跳过文档直接修改

```
"直接修改配置文件，添加这个参数"
```

**正确做法**：
```
"请查阅配置文档，确认这个参数的正确写法和位置，然后再修改"
```

### 错误 3：一次性修改太多

```
"帮我优化所有配置"
```

**正确做法**：
```
"请查阅文档，先帮我优化这一个配置项，成功后再做其他的"
```
