# 安全、风险与是否值得使用

在真正开始配置 OpenClaw 之前: 它确实很强，但它也确实有风险，而且并不是所有人都需要它。

---

## 结论

OpenClaw 的核心能力，是让 AI 助手通过聊天平台来操作你的电脑、文件和任务。

这件事一旦配置得足够开放，能力会很强；但如果配置不当、边界没立住，风险也会非常直接。

---

## 真正要担心的风险

### 1. 文件误删和误改

这是最需要认真对待的风险。

如果你给了它足够大的操作范围，又没有要求它“先给计划和 diff，再执行”，那它就可能：

- 删除你本来不想删的项目文件
- 覆盖已有配置，导致环境直接坏掉
- 误操作到桌面、下载目录、工作资料目录
- 在极端情况下碰到系统相关路径，影响系统正常使用

### 2. 配置越改越乱

OpenClaw 本身自由度很高，但这意味着它很容易进入一种状态：

- 今天加一个 Skill
- 明天改一点 Workspace
- 后天再补一个自动化

最后你自己都不确定：

- 现在到底是哪段配置在生效
- 某个行为是默认行为，还是你之前改出来的
- 问题到底出在 channel、workspace、skill，还是模型本身

### 3. 敏感信息泄露

如果你把 API Key、Token、密码、隐私文档直接放进 Workspace 或聊天环境里，风险会非常高。

不建议把下面这些内容直接交给 OpenClaw 长期管理：

- 各类平台 Token / API Key
- 含敏感信息的原始聊天记录
- 私密附件、客户资料、公司内部文档

### 4. 远程操作带来的放大风险

OpenClaw 的一个卖点，是你可以不在电脑前，也通过 Discord、飞书这类平台远程让它做事。

这同时意味着：

- 你更容易在手机上草率地下指令
- 你不一定能完整看清它给的计划和 diff
- 一旦它开始执行，补救成本可能比本地操作更高

---

## 为什么说“大多数人其实没必要用”

这不是说 OpenClaw 不好，而是说它解决的问题，本来就比较特殊。

如果你只是：

- 想找一个聊天型 AI
- 想要一个写代码助手
- 想偶尔做点简单自动化
- 想总结文档、写写日报、查查资料

那很多更轻量的工具已经够用了，而且门槛更低、风险更小。

OpenClaw 真正适合的，通常是这几类需求：

- 你明确需要“通过聊天平台远程操作电脑”
- 你需要把多个动作串成稳定工作流
- 你愿意花时间维护 Workspace、Skill、配置和权限边界
- 你知道自己希望它长期帮你做哪几件具体的事
