# 场景二：远程代码编写

通过 Discord 或飞书，随时随地远程控制你的电脑完成代码任务。离开工位了？手机也能让 AI 帮你在电脑上改代码、跑脚本、整理文件。

---

## 场景概述

| 项目 | 说明 |
|------|------|
| **适合人群** | 开发者、需要远程操作电脑的用户 |
| **核心能力** | 远程执行代码、文件读写、脚本运行、Git 操作 |
| **使用设备** | 手机、平板、任意能打开聊天软件的设备 |
| **安全机制** | 可限制操作目录、需要确认的操作会先汇报 |

---

## 适合的任务类型

| ✅ 适合 | ❌ 不适合 |
|--------|----------|
| 简单 Bug 修复 | 需要复杂 UI 交互的调试 |
| 添加功能函数 | 实时性要求极高的任务 |
| 生成和整理文档 | 大规模重构（建议在电脑前操作） |
| 运行测试脚本 | 需要频繁看图形界面的工作 |
| Git 提交/推送 | 需要精确鼠标点击的操作 |
| 查找和处理文件 | |

---

## 第 1 步：配置 Coding Skill

```
"请查阅 /docs/skills/ 下关于 Coding 的文档，帮我配置 Coding Agent，让我可以通过 Discord 远程写代码"
```

配置完成后，你可以限制 AI 只能操作特定目录：

```
"配置安全限制：AI 只能访问 ~/projects/ 目录，不能访问其他目录"
```

---

## 第 2 步：基本的远程操作

### 查看文件内容

```
"帮我读取 ~/projects/myapp/src/main.py 的内容"
```

### 修改代码

```
"帮我在 ~/projects/myapp/src/utils.py 里添加一个函数：
接受一个列表，返回去重并排序后的结果，函数名叫 dedupe_and_sort"
```

### 运行脚本

```
"帮我在 ~/projects/myapp/ 目录下运行 pytest，告诉我测试结果"
```

### Git 操作

```
"帮我查看 ~/projects/myapp/ 的 git status，
然后把所有改动提交，commit 信息是 'fix: 修复用户列表排序问题'"
```

---

## 第 3 步：案例 - Obsidian 笔记整理

配置好 Obsidian 集成后，可以随时保存网页内容到笔记：

```
"请帮我：
1. 读取这个链接的内容 https://example.com/article
2. 用中文总结成 200 字以内的笔记
3. 保存到 ~/Obsidian/收藏/ 目录
4. 文件名用文章标题，格式：YYYY-MM-DD-标题.md
5. 在文件开头添加标签：#收藏 #待读"
```

---

## 第 4 步：案例 - 远程修复生产 Bug

收到告警，但人不在电脑前：

```
"~~/projects/api/ 的日志里报错了，
帮我查看最新 50 行日志，找出错误原因，
修复后跑一下测试，没问题就推送到 GitHub"
```

---

## 安全建议

**操作前明确范围**：
```
"在修改文件之前，先告诉我你打算做什么，等我确认再动"
```

**限制操作目录**：
```
"请只在 ~/projects/ 范围内操作，不要访问其他目录"
```

**重要操作要备份**：
```
"修改这个文件之前，先在同目录备份一份，命名加 .bak 后缀"
```
