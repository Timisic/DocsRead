# 场景一：自动化论文推送

你可以把“找论文、筛关键词、整理摘要、定时推送”变成一个自动化流程，减少每天手工刷 arXiv 的时间。

---

## 本章产出

完成本章后，你应该已经：

- 安装并启用一个论文推送 Skill
- 配好筛选规则、推送频道和定时任务
- 能手动触发并验证推送质量

---

## 前置条件

- 已完成 [安装与部署](./01-installation.md)
- 已完成至少一个聊天平台配置（建议 Discord）
- 已阅读 [核心原则：查阅 Docs](./04-core-principles.md)

---

## 操作步骤

### 第 1 步：选择并安装 Skill

你可以在 [ClawdHub](https://clawdhub.com) 或 GitHub 搜索 `openclaw-skill arxiv`。

#### 对 AI 说的话

```text
请查阅 /docs/skills/ 相关文档，帮我推荐 3 个适合“每日论文推送”的 Skill，并说明区别。
```

```text
请安装我选择的 Skill（名称：arxiv-daily），安装后告诉我安装路径和配置文件位置。
```

---

### 第 2 步：配置推送参数

建议先用“最小可用配置”，跑通后再扩展。

#### 对 AI 说的话

```text
请帮我配置论文推送 Skill：
- 分类：cs.AI, cs.LG
- 包含关键词：LLM, transformer, diffusion, agent
- 排除关键词：survey
- 排序方式：按发布时间倒序
- 每次最多推送：5 篇
- 推送平台：Discord
- 推送频道：#paper-daily
- 推送时间：每天 08:30（Asia/Shanghai）
修改后请给我配置 diff。
```

---

### 第 3 步：手动试跑并验收

#### 对 AI 说的话

```text
请立刻手动执行一次论文推送任务，不要改配置。执行完成后：
1) 给我本次命中的论文数
2) 展示前 2 条推送预览
3) 告诉我是否有报错
```

你应该看到类似结构（日期建议用动态值）：

```text
📄 今日 AI 论文推送 (YYYY-MM-DD)
1. 标题...
   ArXiv: ...
   摘要: ...
   链接: ...
```

---

### 第 4 步：开启定时任务

#### 对 AI 说的话

```text
请把这个论文推送任务设置为每天 08:30 自动执行（Asia/Shanghai），并告诉我任务名称、下次运行时间和停用方法。
```

---

## 验证清单

- [ ] 手动执行可以成功推送
- [ ] 推送内容包含标题、链接、摘要
- [ ] 关键词过滤效果符合预期（误报可接受范围内）
- [ ] 定时任务创建成功，且可查看下次运行时间

---

## 常见问题与优化

- 结果太多：收紧关键词，或减少分类数量。
- 结果不准：提高关键词权重，增加排除词。
- 推送太吵：先推送到测试频道，确认稳定后再切正式频道。
- 想看中文摘要：让 AI 在推送阶段加“翻译 + 3 句总结”。

### 失败案例：任务执行了，但频道没收到消息

排查顺序建议：

1. 确认任务确实运行：查看最近一次任务日志是否成功。
2. 确认频道名/频道 ID 是否配置正确（测试频道与正式频道是否搞混）。
3. 确认 Bot 在目标频道有发言权限（Send Messages / View Channel）。
4. 手动触发一次并让 AI 输出“目标频道、消息条数、错误日志摘要”。

可直接对 AI 说：

```text
论文推送任务已执行但频道没收到消息。
请按“任务日志 -> 频道配置 -> 权限 -> 手动重试”顺序排查，
并给我每一步的结论和下一步建议。
```
